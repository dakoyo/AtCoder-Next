# atcoder-next (atc)

English: [README](README.en.md)

AtCoder Nextは、AtCoderでの競技プログラミングを効率的に行うためのCLIツールです。

## ターミナルデモ
![Screencast](./docs/media/vhs/ja.gif)

インストール後、競技プログラミング用のディレクトリで以下のコマンドを実行するだけで環境構築が完了します。

```bash
# 1. ワークスペースの初期化（言語やテンプレートの設定）
atc init

# 2. AtCoderへのログイン
atc login

# 3. ツールチェーンの自動セットアップ
atc t setup
```

## 機能の特徴

* **対話型CLI**: `@clack/prompts`を使用した、視覚的なプロンプトを採用しています。
* **シームレスなログイン**: ブラウザによる自動ログイン、またはクッキーの直接入力に対応しています。
* **スクラッチ生成**: コンテストID（例: `abc300`）を指定することで、問題のディレクトリ構造、テンプレートファイル、サンプルケースを自動でダウンロードします。
* **迅速なローカルテスト**: サンプルケースに対するコードのコンパイルと実行を行います。問題の実行時間制限はAtCoderから自動で取得されます。
* **スマートな提出とリアルタイムジャッジ**: 提出前にローカルテストを自動実行します。提出後はジャッジステータス（AC/WA/TLEなど）をリアルタイムに確認できます。
* **ツールチェーン診断とセットアップ (`doctor`/`setup`)**: ローカル環境とAtCoderの実行環境におけるコンパイラバージョンの乖離を診断し、環境に合わせたコンパイラのセットアップを支援します。
* **ソースコードバンドラー**: 複数のファイルに分割されたライブラリやローカルヘッダファイルを、提出可能な単一のファイルに結合します。
* **多言語対応**: 標準で英語と日本語をサポートしています。

## クイックスタート

### 1. インストール

npmを介してグローバルにインストールできます。

```bash
npm install -g atcoder-next
```

### 2. ワークスペースの初期化

競技プログラミング用のディレクトリに移動し、以下のコマンドを実行します。

```bash
atc init
```

* 表示言語（英語または日本語）を選択します。
* 問題文をMarkdownとして抽出するかどうかを選択します。
* デフォルトの言語テンプレート（C++, Python, Rust, TypeScript, JavaScript, Cなど）を選択します。

### 3. AtCoderへのログイン

提出機能を利用するには、ログインが必要です。

```bash
atc login
```

* **ブラウザ（推奨）**: ブラウザウィンドウが起動します。AtCoderにログインすると、CLIが自動でセッションをキャプチャします。
* **手動クッキー入力**: デベロッパーツールから取得した `REVEL_SESSION` の値を貼り付けます。

## 基本的な使い方

### 1. コンテスト情報の取得

```bash
atc new <contest_id>
# 例: atc new abc300
```

* 対話形式のチェックリストが表示され、ダウンロードする問題を選択できます。
* `-a` または `--all` オプションを付けることで、プロンプトを表示せずにすべての問題をダウンロードできます。
* 設定で問題文のダウンロードが有効化されている場合は、開催中のコンテストを除き、問題文がマークダウン形式でダウンロードされ、VSCodeのプレビュー機能などで確認することができます。

### 2. テストの実行

問題のディレクトリ内で実行するか、対象の問題を指定して実行します。

```bash
# カレントディレクトリの問題に対してテストを実行
atc test

# 特定の問題を指定してテストを実行
atc test abc300 a
```

### 3. コードの提出

```bash
# カレントディレクトリのコードを提出
atc submit

# 特定の問題を指定して提出
atc submit abc300 a
```

* 提出前にローカルでコンパイルとテストを実行します。テストが失敗した場合は、提出を続行するか確認のプロンプトが表示されます。

## 拡張ツール

### 診断とツールチェーンセットアップ (`doctor` / `setup`)

ローカルのコンパイラバージョンをAtCoderの公式環境に適合させます。

```bash
# ローカル環境の診断
atc tools doctor

# コンパイラツールの自動インストールと設定
atc tools setup
```

> [!NOTE]
> **対応OSと制限事項**
> * `doctor` / `setup` 機能は GitHub Actions による CI 環境（Ubuntu、macOS、Windows）でテストされていますが、ローカル環境ごとの個別の設定（セキュリティ制限、パス設定、複数パッケージマネージャの競合など）によっては、予期しない挙動をする可能性があります。
> * 自動インストールがうまく動作しない場合は、エラーメッセージや `~/.atcoder-next/install.log` に出力されるログを確認の上、手動でツールを導入して `.atcoder-next/settings.json` のビルド/実行コマンドを変更してください。

### ソースコードバンドラー (`bundle`)

ローカルにある複数のコードモジュールを1つのファイルにマージします。（C++, C, JavaScript, TypeScript, Python, Rust に対応）

```bash
# ファイルのインポートを1つに統合
atc tools bundle main.rs -o bundle.rs

# toolsはtで省略可能。-oを指定しなかった場合はmain.bundle.cppとして出力されます。
atc t bundle main.cpp
```

> [!WARNING]
> **バンドラーの制限事項（複数言語共通）**
> 本機能は、ソースコード内のインクルード記述（C++の #include "..."、Pythonの import / from ... import、Rustの mod など）に対して、正規表現を用いた単純な文字列置換を行うものです。（js, tsは内部でesbuildを使用しています）マクロや条件分岐による制御（C++の #ifdefなど）、あるいはコメントや文字列リテラル内部に書かれたインクルード記述といった、言語ごとの高度な文脈解析には対応していません。 複雑なモジュール構成や外部ライブラリをバンドルする際には十分にご注意ください。

## 設定

初期化時に `.atcoder-next/settings.json` が作成されます。
このファイルには、設定されたプログラミング言語、拡張子、コンパイルフラグ、実行コマンドなどの情報が含まれており、用途に合わせて自由に編集できます。

## 重要なガイドライン

AtCoderの規約を遵守し、著作権問題を避けるために以下の点に注意してください。

* **開催中のコンテスト**: 開催中のコンテストでは、Markdownへの問題文抽出機能が自動的に無効化されます。AIに問題を読み込ませる目的などで、この制限を回避しないでください。
* **共有の禁止**: 抽出した問題文をパブリックリポジトリに公開しないでください。
